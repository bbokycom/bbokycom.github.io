A.4: Blend the Images into a Mosaic

To create the mosaics, I first determined a central reference image for each set (usually the middle image). I then computed the homographies from each image to this base image. Using these homographies, I warped every other image into the coordinate frame of the base image.

Then, using the warped correspondence points of each image and the base image correspondences, I calculated an average shift for each image that would align it with the base image. Using these shifts, and the bounding corners of all the warped images, I computed the overall bounding box for the mosaic. I created a blank canvas for the mosaic using those dimensions, and placed the base image in the appropriate location.

For every other image, I placed the warped image onto the mosaic canvas at the calculated shift position relative to the base image. To blend the images and reduce edge artifacts, I used scipy.ndimage.distance_transform_edt to compute a distance transform for each image mask, which gives the distance of each pixel to the nearest zero-valued pixel (i.e., the edge).

Using this as a weight map, I performed weighted averaging of overlapping pixels from different images. For example, if two images overlap at a pixel, I add the pixel values of each image weighted by their distance transform value, and divide by the sum of the weights to get a blended pixel value. This helps make the mosaics look much better by smoothing out transitions between images.

B.1: Harris Corner Detection

Harris Corner Detection is a method to find interest points within an image, based on how "corner" a certain point is. This is calculated by using the local image gradients, and seeing which points change significantly when moved in any direction.

Adaptive Non-Maximal Suppression (ANMS) is applied after Harris detection to make the set of corners more well-distributed. This is done by removing corners that are too close to stronger corners, which makes it so that resulting keypoints are not clustered together, and are strong features.

B.2: Feature Descriptor Extraction

The next step of the pipeline is to extract feature descriptors, which will allow for matching between keypoints across different images. In this project, this is done by taking a 40x40 pixel patch around each keypoint, downsampling it to 8x8, and then normalizing the resulting patch. B.3: Feature Matching

We then do feature matching between image pairs using the extracted descriptors. For each descriptor in the first image, we find the nearest and second-nearest neighbors in the second image based on the sum of squared differences. We then take the ratio of these distances (the intuition is that a good match will be much closer than the second-best match). If this ratio is below a chosen threshold (default was 0.8), we consider it a match and store the corresponding points. B.4: RANSAC for Robust Homography

The final step of autogenerating correspondence points is to apply RANSAC (RANdom SAmple Consensus). This algorithm works by iteratively and randomly selecting a small subset of matches to compute a candidate homography. It then applies this homography to all the matches, and tests how many of those are inliers (i.e, how many matches are aligned well after applying the homography). After a set number of iterations, the homography which had the most points classified as inliers is chosen to be the homography. This final set of inliers is then used to compute the final homography matrix.


The manually stitched mosaics in general show more misalignment and blur compared to the automatic mosaics, and took much longer to align. This is because of the tiring, long, and imperfect human effort involved in selecting correspondence points between the images. In comparison, the automatic mosaics are more consistent and aligned, as the RANSAC algorithm effectively filters outliers and finds the best transformation between image pairs.
